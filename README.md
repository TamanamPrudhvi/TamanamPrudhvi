Git Basics

Common Git Commands:

git init: Initializes a new Git repository in your project folder.

git clone [URL]: Copies a remote repository to your local machine.

git status: Shows the current status of your project, including untracked, modified, and staged files.

git add [file]: Stages changes for commit. Use "git add." to stage all changes.

git commit -m "Message": Records changes with a descriptive message.

git branch: Lists all branches in your repo. Use "git branch [branch name]" to create a new branch.

git checkout [branch]: Switches to a different branch.

git merge [branch]: Merges changes from one branch into the current branch.

git pull: Fetches changes from a remote repository and merges them into the current branch.

git push: Pushes your local commits to a remote repository.

git log: Shows a history of commits.

git remote: Lists remote repositories connected to your project.

git remote add [name] [URL]: Adds a remote repository.

git fetch [remote]: Fetches changes from a remote repository.

git clone: Creates a local copy of a remote repository.

Workflow Example:

Clone a repository using git clone [URL].

Create a branch for your new feature with git branch [branch name].

Switch to your new branch with git checkout [branch name].

Make changes to your code.

Stage changes with git add [file] and commit them with git commit -m "Description"

Push your branch to the remote repository with git push origin [branch name].

Create a Pull Request on platforms like GitHub or GitLab.

Review and discuss the changes with your team.

Merge the changes into the main branch if approved.
